# Что перенести в финальную версию из моей ветки

Привет! Посмотрела твою версию (nornikel_hack) — доменный справочник и тюнинг поиска отличные, они остаются. Но твоя копия ответвилась от моего кода до трёх последних коммитов, поэтому в ней нет части готовых фич. Вот что стоит забрать — всё лежит в моём репозитории (архив nauchny-klubok-src.zip), файлы можно копировать целиком.

## 1. Панель графа без пересечений (замена твоей SVG-карты)

**Файл:** `frontend/src/components/KnowledgeGraphPanel.jsx` (новый, самодостаточный)

- раскладка с разрешением коллизий: 0 пересечений узлов и подписей даже на 35+ узлах (у текущей карты узлы наезжают друг на друга);
- бейдж режима real / hybrid / fallback;
- клик по узлу → панель деталей; hover → подсветка соседей;
- тулбар: Вписать / Сброс / Подписи / Экспорт JSON;
- при падении API показывает демо-граф, а не пустоту.

Подключение: в `views.jsx` в GraphView заменить свой `<svg>` на `<KnowledgeGraphPanel graph={graph} ... />` (см. мой views.jsx, там ~15 строк обвязки).

## 2. Тесты графового pipeline

**Файл:** `backend/tests/test_graph_query.py` (новый, 11 тестов)

Проверяют: валидный ответ на любой запрос, уникальность id, целостность рёбер, пустую БД, виртуальные узлы, извлечение чисел (диапазоны/операторы/единицы), синонимы, golden-запросы. Запуск: `python -m pytest backend/tests -q` → 14 passed. Твой domain_knowledge они не трогают — конфликтов не будет.

## 3. Верификация фактов через реальный API

**Файлы:** `frontend/src/App.jsx` (функция `reviewFact`), `frontend/src/components/views.jsx` (CurateView с prop `onReviewFact`)

У тебя вкладка «Проверка» — локальная заглушка, решения пропадают. У меня кнопки «Подтвердить/Отклонить» бьют в существующий `POST /api/facts/{id}/review` — решение пишется в базу + аудит-лог. Это прямое требование кейса «модель верификации знаний».

## 4. Фильтр «отечественная / зарубежная практика»

**Файлы:** `App.jsx` (state `langFilter` + `currentFilters()`), `views.jsx` (переключатель «Практика: Вся/Отечественная/Зарубежная»)

Требование кейса «различать отечественную и зарубежную практику». Backend уже поддерживает `filters.language` — у тебя это просто не выведено в UI.

## 5. Автопоиск экспертов после ответа

**Файл:** `App.jsx` — после `runAnswer` вызывается `runExpertSearch(query, {silent:true})`. Требование «показ связанных экспертов по теме запроса» — эксперты появляются сами, без отдельного поля.

## 6. Чистка текстов UI (важно для демо!)

**Файлы:** `views.jsx`, `layout.jsx`, `constants.js`, `main.py`

- убраны «Copilot», «Evidence-first», «Research gaps», «demo-ready», «Prefer left/right», «needs review» → нормальный русский;
- уверенность и релевантность показываются процентами («Уверенность 68%», а не «0.68»);
- предупреждения backend переведены («Языковая модель недоступна — использован упрощенный режим»);
- **багфикс в `constants.js` (mojibakeScore):** старая эвристика ломала нормальные слова на «Р»/«С» («Россия и мир» превращалось в «>AA8O 8 <8@»). Забери мою версию функции обязательно, у тебя этот баг есть.
- **багфикс в `llm.py` (_readable_snippet):** в ответах больше не показывается сырой JSON из каталогов — вместо него читаемые названия материалов/методов.

## Порядок переноса (чтобы ничего не сломать)

1. Скопировать целиком: `KnowledgeGraphPanel.jsx`, `test_graph_query.py`.
2. Скопировать целиком: `constants.js`, `layout.jsx` (у тебя нет своих правок в них).
3. `views.jsx`, `App.jsx` — у тебя свои правки в GraphView: проще взять мои файлы и вернуть туда твой пропс `query` в GraphView (одна строка).
4. `llm.py`, `main.py` — у тебя свои правки (domain_knowledge): НЕ копировать целиком, перенести точечно `_readable_snippet` + русские тексты предупреждений.
5. Прогнать: `python -m pytest backend/tests -q` и `npm --prefix frontend run build`.
